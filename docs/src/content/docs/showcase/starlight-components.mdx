## Asides

<Aside>
    Default asides work well for neutral notes that should be visible without becoming the visual center of the page.
</Aside>

<Aside type="tip" title="Theme QA" icon="star">
    Keep this page open while adjusting tokens. The fastest way to catch a mismatch is to see every state together.
</Aside>

<Aside type="caution" title="Contrast check">
    Review light and dark mode after changing `--foreground`, `--muted-foreground`, `--border`, or `--code-background`.
</Aside>

<Aside type="danger">
    Do not judge a docs theme only by headings and paragraphs. Tabs, file trees, badges, and code frames need the same care.
</Aside>

## Steps

<Steps>
1. ### Install the plugin

    Add `lucode-starlight` to Starlight's `plugins` array.

2. ### Extend the schema

    Add `ExtendDocsSchema` if you want typed splash hero layouts and announcements.

3. ### Tune the tokens

    Override CSS custom properties from your app stylesheet.

4. ### Inspect the showcase

    Review this page and the typography page before shipping a custom palette.
</Steps>
