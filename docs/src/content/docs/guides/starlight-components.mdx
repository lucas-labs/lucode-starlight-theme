## Asides

<Aside>
    The default aside is useful for secondary context that should stay visible but not dominate the
    page.
</Aside>

<Aside type="tip" title="Theme QA" icon="star">
    Keep a page like this around while building a theme. It makes regressions obvious across
    spacing, colors, and states.
</Aside>

<Aside type="caution" title="Hover states matter">
    Subtle differences between default, hover, and active states are easy to miss unless they are
    shown together.
</Aside>

<Aside type="danger">
    Avoid shipping a theme after only checking headings and code blocks. The long tail of components
    is where mismatches hide.
</Aside>

## Steps

<Steps>
1. ### Open the preview page

    Open this page after changing theme variables or component overrides.

2. ### Compare component states

    Check cards, asides, badges, code, tabs, and link buttons for spacing or contrast regressions.

3. ### Verify navigation polish

    Review the sidebar hover state, active item treatment, and bottom pagination links before
    shipping.

4. ### Ship the theme

    Publish once every built-in component still feels like part of the same design system.
</Steps>
